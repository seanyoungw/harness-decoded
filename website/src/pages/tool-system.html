Every tool call passes through a seven-step gauntlet: lookup, schema validation, permission check, approval gate, sandboxed execution, error classification, and audit logging. None of these steps can be skipped.

Execution path

Every tool call: 7 steps

This is the single choke point that makes the harness safe. Each step can abort execution before the next begins.

1
Tool lookup
Does this tool exist in the registry? Hard fail if not — the model hallucinated a tool name.
if name not in self._tools:
  raise KeyError(f"Unknown tool: {name}")
2
Input schema validation
Args validated against JSON Schema before the tool's execute() is called. The model cannot pass a malformed path to a file tool.
jsonschema.validate(args, tool.input_schema)
# raises ValidationError if args don't match
3
Permission check
tool.required_permissions must be a subset of session.permissions.granted. Missing permissions → PermissionError, surfaced to agent.
missing = required - granted
if missing:
  raise PermissionError(f"Missing: {missing}")
4
Approval gate
Destructive patterns (rm -rf, git push --force, DROP TABLE) trigger a pause. User sees the exact command and approves or denies before execution.
if is_destructive(args):
  approval = await request_approval(tool, args)
  if not approval.granted: raise PermissionError
5
Sandboxed execution
Tool runs with: cwd restriction, env allowlist (no secrets leaked), hard timeout, output size cap. For shell: no shell=True, explicit env dict.
proc = await asyncio.create_subprocess_shell(
  command, cwd=sandbox.cwd, env=allowed_env,
  stdout=PIPE, stderr=STDOUT
)
6
Error classification
Not all errors are equal. RETRYABLE (connection drop) vs INPUT_INVALID (bad args) vs FATAL (corrupted state). Agent loop makes informed recovery decisions.
# RETRYABLE → try again
# INPUT_INVALID → reformulate args
# PERMISSION_DENIED → escalate or abandon
# TIMEOUT → try simpler approach
7
Audit log entry
Written before returning the result. Append-only JSONL. Includes: session_id, iteration, tool, full args, result summary, error kind, duration_ms, approval state.
audit.write(AuditEntry(
  session_id, iteration, tool=name,
  args=args, result_summary=output[:500],
  duration_ms=elapsed, approved=approval
))

Tool catalog

40+ tools, six categories

Each tool is a typed object, not a function decorator. Schema, permissions, and error handling are part of the type, not scattered across call sites.

Undercover Mode

Stealth commits for open-source

When the agent detects a public repository, undercover.ts activates and scrubs all git operations of internal information.
